# Posicionamiento del repo en GitHub 📈

Registro de lo aplicado para ganar visitas y colaboradores. Todo esto se configura
en **Settings** del repo y en la portada (no vive en el código).

## Descripción (About)
> 🦗 Convierte todos tus proyectos en un terrario de artrópodos: cada área es una
> colonia (con reina) y cada proyecto un bicho. Panel HTML estático, SVG
> paramétrico, sin build — el estado se lee en el dibujo.

**Website:** https://chufladizo.github.io/arthropod-terrarium/

## Ajustes recomendados
- ✅ Subir **imagen social** (`assets/social-preview.jpg`) en Settings → Social preview.
- ✅ Marcar "Include in the home page / Releases, Packages" según convenga.
- ✅ Activar **Discussions** (Settings → Features) para ideas abiertas.
- ✅ Crear las etiquetas `design` y `good first issue` (si no existen).
- ✅ Publicar los [retos de diseño](RETOS-DISENO.md) como issues `good first issue`.
- ✅ Fijar (pin) 2-3 issues de diseño para que se vean primero.

## Difusión (opcional)
- Compartir la demo en r/dataisbeautiful, r/webdev, Hacker News «Show HN».
- Un GIF corto navegando el terrario tira mucho en redes.
